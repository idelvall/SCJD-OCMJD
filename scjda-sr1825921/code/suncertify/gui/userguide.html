Introduction

This application, allows the CSRs to show a list of hotel accomodations that match a customer's criteria and book a selected hotel room. All hotel related information is stored in a custom URLyBird 1.2.1 database. Both local and remote databases are supported.

Two types of scenarios are possible:

  1. Standalone: One single application runs: a client application accessing directly a local database. No networking is performed.
  2. Distributed: A server application runs and one or more remote client applications connect to it. Communication is made over the network. This mode should be used in case of several CSRs share a single database.
Therefore three working modes are supported: Standalone client using a local database file, online client using a remote database server, and database server publishing a database over the network.

Launching

Application working modes:
  • Standalone client
    • Uses a local database with no networking involved. From the command line type: 
    java -jar <path_and_filename> alone

  • Networked client
    • Connects to a published server. From the command line type: 
    java -jar <path_and_filename>

  • Server
    • Publishes a database over the net. From the command line type: 
    java -jar <path_and_filename> server

Client application

Client application lets a CSR inspect an URLyBird database and book rooms, updating the database accordingly. Local and remote databases are supported.

  • Opening a local database
    • From the menu bar select File->Open to open the file select dialog, and pick the database file. Then, if everything goes fine, all the rooms stored in the file will be showed in the results table.

  • Closing a local database session
    • If there is a local database session active go to File->Close to end it.

  • Connecting to a remote database server
    • From the menu bar select File->Connect to open the connect dialog. The remote database must be previously published over the network by a Server application. Two connection parameters are needed:
    • Server address: The host name or IP address of the machine where the server is running. Identifies the server machine in the network.
    • Port: This numeric value must be the same that was specified on server startup. This parameter identifies the server process within the server machine.

  • Disconnecting from a remote database server
    • If a remote database session is active go to File->Disconnect to end it.

  • Searching hotel rooms
    • Once a database session is opened (local or remote), searches can be performed:
    • Search all rooms: From the menu bar select Search->View all database, or press the following key combination Ctrl+D.
    • Search dialog: This dialog allows the user to search the database for all rooms, or for rooms where the name and/or location exactly match values specified by the user. To open this dialog, from the menu bar select Search->Search, or press the following key combination Ctrl+F.

  • Refreshing the results view
    • In remote environments, the results view in client applications often become stale due to modifications made in the database by other users. Refreshing the current view is repeating the last search made. To refresh the view, from the menu bar select File->Refresh, click the Refresh button, or press the key F5.

  • Booking a room to customer
    • Rooms not belonging to a customer can be booked with the client application. To book a room, select it (clicking on it with the mouse or navigating with the arrow keys), and then press Enter or click the Book button. Then the book dialog will be shown, asking to enter the customer ID. While this dialog is open, the room is locked and only that CSR can book the room. On the other hand, is possible that the selected room is being booked by other CSR, if this case occurs, the client application will be bloked until the locked room has been released.
    Tip: Use the Left arrow and Right arrow to navigate only through available rooms.

  • Exiting the application
    • From the menu bar select File->Exit to quit the application.

Server application

Server application publishes a database over the network, so it can be shared by multiple remote clients. For server launching information see this section.
  • Starting the server
    • To start the server the following mandatory parameters must be specified:
    • Database file: Enter the path of the database file to use, or press the ... button to select a database file
    • Server port: Enter a numeric value. This number must be known by the clients so they can later connect to the server. The port must not be used by other server application running in the same machine.
    Then press the Start button to start the server.

  • Closing the server
    • Press the Exit button, or from the menu bar select File->Exit to close the server. If the server has been started, the shutdown process takes some time to finish.

Troubleshooting

  • Reset to defaults
    • Delete the suncertify.properties file generated in application starting path.

  • Logs
    • Both server and client applications generate log files with detailed executions history information. Log path location is specified in server and client GUIs.
From the menu bar select Help->Troubleshooting for detailed path information.
